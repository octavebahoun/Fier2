# Accessibility Review — FIERI Research

## Verdict
needs work

## Summary
The spine provides a solid foundation with well-specified FAQ accordion ARIA, focus visible indicators, and a stated WCAG AA contrast goal, but leaves critical gaps across form accessibility, modal focus trapping, skip navigation, touch targets, and screen reader support for dynamic content and decorative elements. Ten of the ten accessibility areas assessed have gaps; four findings are rated critical.

## Findings

- **[critical]** No skip-to-content link specified. *Spec gap:* keyboard users have no way to bypass repeated navigation blocks. *Fix:* add a `Skip to main content` link as the first focusable element; specify `href="#main-content"`, visible on focus via `:focus-visible` with brand-violet-light ring, and hidden off-screen by default.
- **[critical]** No focus trapping in modals/dialogs. *Spec gap:* when a modal opens, Tab can escape the modal, trapping the user behind the backdrop or onto background content. *Fix:* specify focus trap behavior — on modal open, move focus to first focusable element; on Tab from last element, cycle back to first; on Escape, close and return focus to the trigger element. Specify `data-focus-trap` or `FocusTrap` component usage.
- **[critical]** Form patterns lack labels, `aria-invalid`, and error announcements. *Spec gap:* no `<label>` elements or `aria-label` specified for any input; error messages have no `role="alert"` or `aria-describedby` connection, so screen readers won't announce validation errors. *Fix:* mandate `<label htmlFor="id">` for every input; on validation error, set `aria-invalid="true"` and link the error message via `aria-describedby`, wrapping it with `role="alert"`.
- **[critical]** Touch target minimum size not documented. *Spec gap:* on mobile (< 768px), small controls (Lucide icons in the collapsed 40px sidebar, badge circles of 20px, inline newsletter button) may fall below the 44×44px WCAG 2.5.8 minimum. *Fix:* add a touch-target guideline: all interactive elements must have a minimum hit area of 44×44 CSS pixels on mobile; for icon-only buttons, apply transparent `::before`/`::after` extension or `min-w-[44px] min-h-[44px]` padding.
- **[high]** `prefers-reduced-motion` handling incomplete. *Spec gap:* only the ticker and animated counter are covered; the Bento Grid hover lift (200ms), sidebar expand (300ms), route-transition fade (200ms), ScrollToTop fade+scale, carousel slide, pulse dot (2s loop), and button 1px hover translate are all unmentioned. *Fix:* audit every animation in "Component Patterns" and "Interaction Primitives" and pair each with a reduced-motion rule (e.g., `transition: none`, `animation: none`, or immediate-jump in `useReducedMotion()`).
- **[high]** No `role="dialog"`, `aria-modal`, or `aria-labelledby` on modals. *Spec gap:* the glassmorphism inscription modal and any other dialogs have no ARIA dialog semantics. *Fix:* specify `role="dialog"`, `aria-modal="true"`, and `aria-labelledby` pointing to the modal title for every overlay component.
- **[high]** Navigation landmarks missing ARIA roles. *Spec gap:* Navbar and Sidebar have no landmark roles, and the 13 named sections (`hero`, `faq`, etc.) have no `aria-labelledby` or region roles. *Fix:* specify `<nav aria-label="Navigation principale">` for the top bar, `role="complementary"` or `<aside aria-label="Menu chercheur">` for the sidebar, and `role="region"` with `aria-labelledby` referencing each section heading.
- **[high]** Color contrast not systematically verified for all token pairs. *Spec gap:* the spec states WCAG AA compliance but only checks specific pairs (Violet Royal, Orange Brûlé) and notes Ambre Premium (`#FF8A3D`) fails for small text. It does not verify combinations like `text-secondary-light: #6B7280` on `bg-tertiary-light: #EAF0FF` (approx. 4.1:1 — marginal). *Fix:* add a contrast matrix in DESIGN.md that tests every foreground/background token pair from the color palette and flags any failing combos; set a hard minimum of 4.5:1 for body text and 3:1 for large text.
- **[high]** No `lang` or `dir` attribute specified. *Spec gap:* the HTML root element language and direction are not documented. *Fix:* add a rule: `<html lang="fr" dir="ltr">` on every page.
- **[high]** Logical Tab order not specified. *Spec gap:* while Tab navigation is mentioned for the FAQ, no explicit Tab order (or caveat against positive `tabindex`) is documented for the overall page flow. *Fix:* specify that focus order must follow DOM order; never use positive `tabindex` values; the Tab sequence should be: skip-link → Navbar → main content → footer.
- **[medium]** Carousel missing arrow-key navigation and ARIA roles. *Spec gap:* the "Carrousel des Membres Actifs" has swipe on mobile and arrow buttons on desktop but no left/right arrow-key handling or `role="region"`/`aria-roledescription="carousel"`. *Fix:* add keyboard support: Left Arrow / Right Arrow to navigate slides; specify `role="region"` and `aria-label="Membres actifs"` on the carousel container.
- **[medium]** Decorative icons and dynamic content lack screen reader attributes. *Spec gap:* Lucide Icons in the collapsed sidebar, the pulsating dot, and the `✦` glyph are not annotated with `aria-hidden="true"`; toast messages and skeleton loading states have no `aria-live` region. *Fix:* add `aria-hidden="true"` to all purely decorative SVGs and pseudo-element glyphs; wrap toasts in `role="status" aria-live="polite"`; wrap skeleton containers in `aria-label="Chargement..." aria-busy="true"`.
- **[low]** Accordion pattern solid but could document animated region disclosure. *Spec gap:* the FAQ accordion uses `aria-expanded` and `aria-controls` correctly but does not specify `aria-hidden` on collapsed panes or handling for `aria-roledescription`. *Fix:* add that collapsed panels should have `aria-hidden="true"` (or rely on `hidden` attribute) and expanded panels `aria-hidden="false"`.

## Coverage

- **Covered:** FAQ accordion ARIA (`aria-expanded`, `aria-controls`, `role="region"`, `aria-labelledby`); focus visible indicator (brand-violet-light, 2px offset, `:focus-visible`); WCAG AA as a stated goal (partial); reduced motion for infinite ticker and animated counter; `aria-label="Retour en haut"` on ScrollToTop; keyboard shortcuts (`⌘K`, `g+h`, `g+d`, Escape, Tab, Enter/Space); form validation with inline error messages and `type="email"`.

- **Gaps:** skip-to-content link; modal focus trapping; form labels and `aria-invalid`/`aria-describedby`; touch target minimum size (44×44px); reduced motion for remaining 6+ animation types; dialog ARIA roles (`role="dialog"`, `aria-modal`); navigation landmark ARIA roles; systematic color-contrast verification of all token pairs; `lang`/`dir` attributes; logical Tab order specification; carousel arrow-key navigation; decorative icon `aria-hidden`; `aria-live` regions for dynamic content/toasts/skeletons.
